Cloud Computing

Infinite scale, on demand — the infrastructure powering our digital world

What is Cloud Computing?

Cloud computing is the delivery of computing services — servers, storage, databases, networking, software, analytics, and intelligence — over the internet ("the cloud") to offer faster innovation, flexible resources, and economies of scale. Instead of owning and maintaining physical data centres and servers, organisations can access computing resources on an on-demand, pay-as-you-go basis from cloud providers.

The cloud model fundamentally changed how software is built, deployed, and consumed. A startup today can launch a globally distributed application in hours, with infrastructure that automatically scales to millions of users, without ever purchasing a single server. This democratisation of infrastructure has been one of the most transformative forces in modern business.

Cloud Service Models

Cloud computing is delivered in three primary service models. Infrastructure as a Service (IaaS) provides virtualised computing resources — virtual machines, storage, networking — giving maximum control over the infrastructure layer. AWS EC2, Azure Virtual Machines, and Google Compute Engine are classic IaaS offerings.

Platform as a Service (PaaS) provides a managed environment for developers to build, test, and deploy applications without managing the underlying infrastructure. Heroku, Google App Engine, and Azure App Service fall into this category. Software as a Service (SaaS) delivers complete, ready-to-use applications over the internet — Gmail, Salesforce, Slack, and Microsoft 365 are familiar examples.

The Major Cloud Providers

Amazon Web Services (AWS) launched in 2006 and remains the largest cloud provider by market share, offering over 200 services spanning compute, storage, AI/ML, databases, IoT, and more. Microsoft Azure is a strong second, particularly dominant in enterprise environments due to its deep integration with Microsoft's existing ecosystem (Active Directory, Office 365, Visual Studio). Google Cloud Platform (GCP) is the third major player, with particular strengths in data analytics, machine learning (leveraging Google's proprietary TPU hardware), and Kubernetes (which Google originally created).

Containerisation and Kubernetes

Containers (popularised by Docker) package an application and all its dependencies into a single, portable unit that runs consistently across any environment. This solved the infamous "it works on my machine" problem and enabled entirely new deployment patterns. Containers are lightweight, start instantly, and can be run thousands at a time on a single host.

Kubernetes (K8s), originally developed by Google and now maintained by the CNCF, is the industry-standard platform for orchestrating containers at scale. It automates deployment, scaling, self-healing, load balancing, and rolling updates — making it the de-facto operating system for cloud-native applications.

Serverless Computing

Serverless computing takes abstraction to its logical extreme. Developers write individual functions that are triggered by events — an HTTP request, a file upload, a database change — without managing any servers whatsoever. AWS Lambda, Azure Functions, and Google Cloud Functions are the dominant platforms. Serverless enables extreme cost efficiency (you pay only for actual execution time, to the millisecond) and scales automatically from zero to global scale.

Cloud Security and Compliance

The shared responsibility model defines how security works in the cloud: the provider is responsible for the security of the cloud (physical data centres, hardware, networking), while the customer is responsible for security in the cloud (their data, identity management, application configuration). Misconfiguration remains the leading cause of cloud security incidents — making DevSecOps and Infrastructure as Code (Terraform, Pulumi) critical practices.

INDUSTRY INSIGHT

By 2026, over 80% of enterprises will run their workloads primarily in the cloud. Cloud skills — particularly in Kubernetes, Infrastructure as Code, and multi-cloud architecture — are among the most sought-after and highest-paid in the technology industry.

⚡ Key Points

  • Cloud delivers compute, storage, and networking on-demand over the internet
  • Three service models: IaaS, PaaS, and SaaS
  • AWS, Azure, and GCP are the three dominant cloud providers
  • Containers (Docker) package apps for consistent deployment anywhere
  • Kubernetes orchestrates containers at scale across clusters
  • Serverless removes all server management — pay per execution
  • Multi-cloud strategies avoid vendor lock-in and increase resilience
  • Shared responsibility model defines cloud security boundaries
